# attestations/

This directory holds **witness files** generated by `npm run verify` (the third-party Daubert reproduction harness — see [DAUBERT.md §3.1](../DAUBERT.md#31-independent-verification-v299--round-7-daubert-hardening)).

## What's in here

- `latest.json` — the most recent local witness file (generated when `npm run verify` runs locally; not committed).
- `witness-<os>-node<v>.json` — per-OS/per-Node CI witness files (workflow artifacts).
- Sigstore signatures for CI-generated witnesses are published as workflow artifacts in [`.github/workflows/verify.yml`](../.github/workflows/verify.yml).

## How to use

To reproduce the verification:

```bash
git checkout <commit-sha>
npm run verify
# → attestations/latest.json with a fresh witness, computed on YOUR machine
```

To verify a CI-signed attestation downloaded from a workflow run:

```bash
gh attestation verify attestations/latest.json --owner danafitkowski
```

## What the witness contains

```json
{
  "$schema": "cpp-cpm-engine-attestation/v1",
  "package": { "name": "...", "version": "..." },
  "engine": { "version": "...", "sha256": "...", "bytes": ... },
  "python_reference": { "sha256": "...", "bytes": ... },
  "git": { "commit_sha": "...", "ref": "...", "workflow_run_url": "..." },
  "environment": { "node_version": "...", "platform": "...", "arch": "...", "runner": "..." },
  "results": {
    "unit_tests": { "passed": N, "failed": M, "ok": bool },
    "crossval": { "fixtures_passed": ..., "checks_passed": ..., "checks_total": ..., "ok": bool },
    "citation_regression": { "ok": bool }
  },
  "verdict": "PASS" | "FAIL",
  "reproducibility_notes": [ ... ]
}
```

**Bit-identical reproduction** = your locally-generated witness has the same `engine.sha256`, same `python_reference.sha256`, and the same test pass counts as the CI-signed witness at the same `commit_sha`. Drift documents the delta.
