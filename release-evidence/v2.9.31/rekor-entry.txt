  required. Anyone can confirm the entry exists, was signed at the
  documented timestamp, and matches the certificate subject above.

Option 2 — Rekor CLI:

  rekor-cli get --log-index 1616950323

Option 3 — `gh attestation verify` (recommended; pulls the full
verification chain):

  gh attestation verify release-evidence/v2.9.31/witness-v2.9.31.json \\
      --owner danafitkowski --format json

  The full captured output is in sigstore-attestation-output.txt

## v2.9.31 vs v2.9.30 — what changed under the chain

The engine source SHA-256 changed because v2.9.31 ADDED the Section Q
Forensic Strict Mode code (computeCPMForensicStrict, StrictForensicViolation,
FATAL_STRICT_CONTEXTS, _applyForensicStrictValidation, runCPM strict-mode
refusal). The non-strict path is byte-identical in behavior to v2.9.30 —
strict mode is additive validation, not a math change.

  Engine SHA-256 v2.9.30:  30c633e2c5850cfe24e4a42c146f62095295f5ac7193b04b10d37bde50c7ee94
  Engine SHA-256 v2.9.31:  a6b9e8d93f156b4af487082c15e4141236a93f20e9bfa90b57a95521309ac7fe

  Python ref SHA-256 (both v2.9.30 and v2.9.31):
                            50ddea54d9098395199e808a037b4dde70b13e1373db79bcf12957c05e80d8d7
  (unchanged — strict mode is JS-side validation; Python reference engine
   continues to mirror the non-strict Section C math.)
